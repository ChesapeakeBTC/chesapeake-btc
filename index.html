🎓 Bitcoin 101: UTXOs

Fact of the Day: Bitcoin doesn't track balances. There's no account ledger somewhere that says "this address holds 0.5 BTC." Instead, the network tracks unspent transaction outputs, or UTXOs. Every time you receive bitcoin, a transaction creates an output locked to your key. That output sits in the global UTXO set until you spend it. When you do spend, your wallet selects one or more UTXOs as inputs, consumes them entirely, and creates new outputs: one for the recipient and typically one back to yourself as change. Think of it like paying with physical bills. If you owe someone $7 and hand over a $10, you get $3 back. The $10 bill is gone, replaced by new ones. UTXOs work the same way. Your wallet's "balance" is just the sum of all the unspent outputs your keys can unlock. Right now, the Bitcoin UTXO set contains roughly 190 million entries, and every full node keeps a copy to validate new transactions instantly.

Action Item: Go to mempool.757btc.org and click on any recent transaction. Look at the inputs on the left and the outputs on the right. You'll see exactly which UTXOs were consumed and which new ones were created. If there are two outputs, one is probably the payment and the other is change going back to the sender. Understanding this structure is how you start reading the blockchain like a ledger instead of a mystery.
Source: Learn Me a Bitcoin — UTXOs

🌐 The Wider Network

Freedom Tech
If you contribute to any open-source project, the Bitcoin 2026 conference is offering free Open Source Passes and the application deadline is this Monday, April 20. The pass gets you full conference access at no cost. Applications are reviewed weekly, so submitting early improves your chances. The conference hasn't announced final dates yet, but the pass program has been running since January. Open-source work is the backbone of Bitcoin's development. If you've written code, filed bugs, reviewed PRs, or maintained documentation for any FOSS project, you qualify.
Source: Bitcoin 2026 — Open Source Pass

Product Update
BTCPay Server shipped v2.3.7, the first release built on .NET 10. Beyond the framework upgrade, there are some practical improvements: top-up invoices now generate amount-less BOLT11 invoices instead of LNURL when the Lightning provider supports it, the checkout page gained RTL language support for Arabic, Hebrew, and Farsi, and wallet transactions can now carry comments directly in the send view. If you run a BTCPay instance, update your plugins after upgrading — the .NET 10 migration may require plugin developers to adjust their builds.
Source: GitHub — BTCPay Server v2.3.7
